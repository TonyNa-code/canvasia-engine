# Canvasia Engine 创作者零代码使用教程

这份教程写给第一次接触 Canvasia Engine 的创作者。目标不是让你先学编程，而是先跑起来、点起来、做出一小段能试玩的视觉小说 Demo。

当前版本仍是 Preview / Early Access，适合试做项目、体验流程、反馈问题和做小型原型。正式发布作品前，建议完整跑一遍“发布前检查”和导出包里的验收清单。

## 一句话版

1. 先打开 GitHub 仓库：[https://github.com/TonyNa-code/canvasia-engine](https://github.com/TonyNa-code/canvasia-engine)
2. 想直接体验就去 Releases 下载对应系统的编辑器包。
3. 如果暂时没有你系统的编辑器包，就用源码方式启动。
4. 打开编辑器后，从“项目中心”新建可试玩 Demo，或按需新建空白项目。
5. 替换或上传背景、人物、音乐，写几句自己的台词，点“试玩”看效果。
6. 做完后到“预览导出”生成试玩包或原生 Runtime 包。

## 先下载哪个

打开仓库右侧或页面里的 `Releases`，优先下载和自己系统对应的编辑器预览包。

| 使用场景 | 推荐选择 |
| --- | --- |
| 只想点开体验 | 下载对应系统的编辑器预览包 |
| Release 里没有自己系统的包 | 下载 Source code，用源码方式启动 |
| 想试玩别人导出的游戏 | 下载网页试玩包或原生 Runtime 包 |
| 想确认文件没坏 | 同时下载 `.sha256`、`.checksum.json` 或 verify 脚本进行校验 |

macOS 或 Windows 可能会提示“来源未知”或 SmartScreen，这是未签名 Preview 包常见现象。只建议从官方仓库 Release 页面下载，并优先校验文件。

## 用源码启动

如果没有现成编辑器包，可以用源码启动。需要先安装 Python 3。

macOS / Linux：

```bash
git clone https://github.com/TonyNa-code/canvasia-engine.git
cd canvasia-engine
python3 run_editor.py
```

Windows：

```bat
git clone https://github.com/TonyNa-code/canvasia-engine.git
cd canvasia-engine
py -3 run_editor.py
```

如果 Windows 没有 `py`，可以试：

```bat
python run_editor.py
```

启动成功后，浏览器会打开本地编辑器页面。地址里出现 `127.0.0.1` 是正常的，意思是编辑器在你自己的电脑上运行，不是把项目上传到了陌生网站。

## 第一次打开怎么做

### 第一步：进入项目中心

打开编辑器后，先看“项目中心”。这里可以新建项目、打开已有项目，也可以看到示例项目。

推荐第一次这样做：

1. 点“新建可试玩 Demo”。
2. 给项目起名。
3. 保持新手模式。
4. 先进入试玩页点几下，确认背景、角色、BGM 和第一句台词都能跑起来。

如果你想完全从零搭建，也可以选择“新建空白项目”，再手动创建第一章、首场景和起步骨架。示例项目会单独列在项目中心下方，只有需要参考时再打开。

### 第二步：导入素材

视觉小说最常用的素材大概是这些：

| 素材类型 | 可以放什么 |
| --- | --- |
| 背景 | 教室、街道、房间、天空、CG 场景图 |
| 角色立绘 | 人物表情、姿势、差分图 |
| CG | 事件图、回想图、章节插图 |
| BGM | 日常、紧张、告白、结尾音乐 |
| 音效 | 门声、手机提示、脚步声 |
| 语音 | 角色台词语音 |
| 字体 | 项目自定义字体 |

一般流程是：先上传素材，再到剧情里引用它们。不要一开始就追求素材完整，先用几张图和一首音乐把流程跑通，会更舒服。

如果你有自己的 OpenAI API Key，也可以在“补素材”页使用 `AI 生成素材`：

1. 选择要生成的类型，比如背景、立绘、CG 或界面素材。
2. 写一句提示词，比如“雨后黄昏的校园走廊，视觉小说背景”。
3. 如果想让同一项目画风更统一，可以点击画风预设，或手动填写“画风补充”，比如“清透蓝白、柔和光线”。
4. 如果生成的是立绘，可以选择已有角色和表情；勾选后可同步设为角色默认立绘。
5. 选择尺寸和质量。
6. 点击“生成并导入素材库”。

生成成功后，图片会直接进入当前项目素材库；如果你选择了角色绑定，立绘也会同步写入角色表情。API Key 只用于这一次生成，不会写进项目文件。面板会提示提示词长度，并提前拦截超长画风补充、非法模型名、透明背景 + JPEG 等容易失败的组合。没有 Key 也没关系，普通上传素材和本地模板助手都可以照常使用。

### 第三步：写第一段剧情

可以先按这个最小结构写：

1. 建一个章节，比如“第一章”。
2. 建一个场景，比如“教室黄昏”。
3. 添加背景。
4. 添加角色。
5. 写几句旁白和角色台词。
6. 加一个选项，比如“打招呼”或“保持沉默”。
7. 分别写两条不同后续。

做视觉小说时，建议先把“能从开始走到结尾”的主线跑通，再回头加分支、演出和细节。先让小船下水，再给它装霓虹灯。

需要好感度或道具门槛时，展开某个选项里的“这个选项什么时候出现”：选择“条件不满足时隐藏”可做秘密路线，选择“条件不满足时锁定”则会保留按钮并告诉玩家还缺什么。多条条件会同时判断；建议每组选项至少留一个“始终可选”的保底项，项目巡检也会检查门控缺变量、空规则和无保底路线。

如果你已经在 Word、备忘录或聊天记录里写了一段台本，也可以直接用故事页的“手写剧本转剧情卡片”：

```text
旁白：雨声贴着窗沿落下。
悠奈：你终于来了。
stage image show letter layer note at center front width 58 opacity 92 duration 0.7 easing ease_in_out
等待 0.8
我把伞往她那边递过去。
stage image hide note duration 0.3
- 问她为什么在这里
- 先沉默陪她一会儿
```

点击“预览识别”后，编辑器会把 `角色：台词` 识别成台词卡，把普通句子识别成旁白卡，把 `等待 0.8`、`wait 0.8` 或 `pause 800ms` 识别成等待停顿卡，把连续的 `- 选项` 合并成一个选项卡。`stage image show / update / hide` 会被识别成舞台贴图卡：示例中的 `letter` 是图片素材提示，`note` 是可复用图层名。确认没问题后再点“插入识别结果”，它会插入到当前选中的剧情卡片后面；如果没有选中卡片，就会插到当前场景末尾。

## 常用按钮怎么理解

| 入口 | 用途 |
| --- | --- |
| 项目中心 | 新建、打开、复制、删除项目 |
| 写剧情 | 编辑章节、场景、台词、选项和分支 |
| 补素材 | 管理背景、立绘、CG、BGM、音效、语音、字体 |
| 看角色 | 管理角色资料、立绘和角色相关设置 |
| 查问题 | 查看项目检查、发布前问题和建议修复顺序 |
| 试玩收尾 | 预览当前内容、导出试玩包或 Runtime 包 |
| 新手模式 | 保留常用入口，适合第一次做项目 |
| 高级模式 | 显示更多变量、条件分支和复杂演出能力 |

如果不知道下一步做什么，优先看“项目中心”和“查问题”。它们更像导航台。

## 想做中日英多语言怎么办

如果作品想同时给中文、日文或英文玩家试玩，可以先在“项目设置 / 成品体验设置”里配置默认语言和成品可切换语言。

当前建议流程：

1. 先用最熟悉的语言把主线写完。
2. 在多语言设置里勾选要支持的语言。
3. 给角色名、场景名、台词和选项逐步补翻译文本。
4. 导出后在网页 Runtime 或原生 Runtime 的设置里切换语言检查效果。

缺少某句翻译时，成品会自动回退到默认文本，不会因为漏翻一句就卡住试玩。

## 做出第一个 5 分钟 Demo

建议按下面这个顺序，不要一开始就铺太大。

如果已经从“新建可试玩 Demo”开始，这一步基本已经完成；如果你选择的是空白项目，可以先点“一键生成起步骨架”。它会补第一个角色、背景和 BGM，并尽量把这些基础演出接进首场景；你后面只需要把占位素材替换成正式图片和音乐。

### 1. 做一条最短主线

准备：

- 1 张背景
- 1 个角色立绘
- 1 首 BGM
- 10 到 20 句台词
- 1 个选择题

目标：

- 能从标题页开始游戏
- 能看到角色出场
- 能听到 BGM
- 能点到一个选项
- 能走到一个简单结尾

### 2. 加一点 gal 味

主线能跑后，再加：

- 角色淡入淡出
- 立绘位置和大小调整
- 用舞台贴图卡加入信件、手机、道具、前景或 Cut-in，并设置位置、大小、透明度、旋转和前后层
- BGM 淡入淡出
- 文本框样式
- 角色表情差分
- 一张 CG
- 一个回想馆或图鉴馆条目

### 3. 试玩并修问题

每做一小段就点一次试玩。不要攒到最后才测，不然问题会像共通线一样越走越长。

重点确认：

- 按钮能点
- 台词顺序正确
- 选项能跳到正确后续
- 背景和立绘没有丢
- BGM 没有异常叠在一起
- 存档和读档能恢复；原生 Runtime 的新存档会显示当前游戏画面缩略图，旧存档没有图片也应能正常读取
- 如果作品准备支持手柄，网页 / 桌面试玩包与原生 Runtime 都应连接实机，用摇杆 / 十字键和确认 / 返回键完整走一遍标题页、选项、存档、设置与历史

## 智能创作助手怎么用

编辑器里有 `Canvasia Assistant`。它可以辅助生成剧情、灵感、角色设定和素材提示。

默认模式：

- 不需要 API Key
- 不联网
- 不产生费用
- 适合快速拿灵感

真模型模式：

- 可以使用创作者自己的 OpenAI、DeepSeek、通义千问、Kimi、智谱 GLM 或自定义兼容 API Key
- API Key 不会写进项目文件
- 只在勾选本机记住时保存在当前浏览器
- 不想保存时可以点“忘记本机 Key”

建议用法：

1. 先写自己的剧情大纲。
2. 让助手补几种台词风格。
3. 只挑喜欢的插入项目。
4. 插入前自己读一遍，避免角色语气跑偏。

## 怎么导出给别人试玩

进入“试玩收尾”或“预览导出”页面后，可以选择导出类型。

| 导出类型 | 适合什么 |
| --- | --- |
| 网页试玩包 | 最适合快速发给朋友看，体积轻，容易预览 |
| Windows / macOS / Linux 桌面包 | 更像普通桌面应用 |
| 原生 Runtime 包 | 用于验证脱离 HTML 的桌面播放链，适合进一步测试 |

第一次发给朋友或测试者试玩，推荐先用“网页试玩包”。如果想测试更接近独立 App 的体验，再尝试桌面包或原生 Runtime 包。

导出前建议：

1. 点一次项目检查。
2. 处理红色错误。
3. 看一眼素材体积雷达，先处理特别大的视频、BGM、CG 或未使用素材。
4. 看一眼 Runtime 首屏加载预算，确认入口场景没有过大的 OP、大图、长 BGM 或缺文件。
5. 至少完整试玩一遍开头到结尾。
6. 导出后自己先解压打开一次。
7. 再发给别人。

## 原生 Runtime 包怎么验收

原生 Runtime 包会随包生成一些报告和脚本，不是给新手增加负担，而是为了减少“我这能跑，你那不能跑”的尴尬。

重点看这几个：

| 文件 | 用途 |
| --- | --- |
| `native-runtime-doctor-report.md` | 一键体检总览，先看它能快速知道包结构、存档、设置、演出、3D 和视频基础链路是否正常 |
| `native-runtime-crash-feedback.md` | 打不开或闪退时用的反馈模板；玩家机器上运行 `python3 runtime_player.py --write-crash-feedback-reports .` 后再发给维护者 |
| `native-runtime-release-control-report.md` | 发布总控，告诉你当前包大体是否适合继续发布 |
| `native-runtime-release-acceptance.md` | 发布前人工验收清单，照着勾就行 |
| `native-runtime-file-integrity.md` | 文件完整性报告，确认导出包核心文件没坏 |

如果只是发一个小 Demo，可以先看 Markdown 文件，不一定要研究 JSON。

## 常见问题

### 打开后是浏览器页面，是不是网页游戏？

编辑器本体目前使用浏览器界面是为了降低门槛。项目和素材仍在本机处理。后续导出的游戏可以是网页试玩包，也可以是桌面包或原生 Runtime 包。

### 为什么地址是 127.0.0.1？

`127.0.0.1` 代表你自己的电脑。编辑器服务在本机运行，浏览器只是显示界面。

### macOS 提示打不开怎么办？

Preview 包可能未签名。请确认文件来自官方 Release 页面。确认后可以右键应用，选择“打开”，再按系统提示继续。

### Windows 提示 SmartScreen 怎么办？

Preview 包可能触发 SmartScreen。请先确认来源和校验文件，再选择继续运行。不确定来源时不要运行。

### 点了按钮没反应怎么办？

先刷新页面。如果仍然不行，可以在 GitHub Issues 反馈，并说明：

- 使用系统
- 浏览器
- 点了哪个按钮
- 预期发生什么
- 实际发生什么
- 最好附截图

### 素材不显示怎么办？

先检查素材是否成功导入，再确认剧情或角色里是否选择了正确素材。导出前也可以运行项目检查，看有没有缺失素材提示。

### BGM 或语音没声音怎么办？

先确认系统音量、浏览器音量和编辑器内音量设置。部分浏览器需要用户先点击页面，音频才会开始播放。

### 可以商用自己做的游戏吗？

可以使用本引擎制作并商业发布自己的游戏。引擎源码和引擎本体的再分发、改名售卖、商用封装等行为，请遵守仓库里的 Creator License 1.0。

## 给新用户的最短体验路线

如果只是想快速看看它能不能用，可以按这个路线：

1. 下载或启动编辑器。
2. 打开项目中心。
3. 新建可试玩 Demo。
4. 点进试玩页，确认第一段剧情能跑起来。
5. 回到编辑器，把占位背景、立绘、BGM 和台词替换成自己的内容。
6. 加一个选项或分支。
7. 再试玩一遍。
8. 没问题后导出网页试玩包或桌面 Runtime 包。

跑完这 8 步，就已经从“看看项目”进入“我做了一个小 Demo”的世界线了。
